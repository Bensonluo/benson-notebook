## DDD: Domain Driven Design

- 不要数据驱动或界面驱动

- 微服务要界限清晰， 职能清晰的分层， 控制合适的粒度而不是过度的拆分

- 四层架构:  Interface -> Application -> Domain -> Infrastructure



## Go-micro 基础
